# Session 内存存储

## 目的
此目录存储基于会话的内存数据、对话历史和使用 Claude-Flow 编排系统的开发会话上下文信息。

## 结构
会话按日期和会话 ID 组织，便于检索：

```
memory/sessions/
├── 2024-01-10/
│   ├── session_001/
│   │   ├── metadata.json        # 会话元数据和配置
│   │   ├── conversation.md      # 完整对话历史
│   │   ├── decisions.md         # 关键决策和理由
│   │   ├── artifacts/           # 生成的文件和输出
│   │   └── coordination_state/  # 协调系统快照
│   └── ...
└── shared/
    ├── patterns.md              # 通用会话模式
    └── templates/               # 会话模板文件
```

## 使用指南
1. **会话隔离**: 每个会话获得自己的目录
2. **元数据完整性**: 始终填写会话元数据
3. **对话记录**: 记录所有重要的交互
4. **Artifact 组织**: 清晰地结构化生成的文件
5. **状态保留**: 定期快照协调状态
